# dotfiles

My personal Hyprland dotfiles on Debian, built on top of [JaKooLit's Debian-Hyprland](https://github.com/JaKooLit/Debian-Hyprland) installer.

## System

- **OS:** Debian (sid/testing)
- **WM:** Hyprland (Wayland)
- **Shell:** Zsh + oh-my-zsh
- **Monitors:** Acer XV275K P3 (4K 160Hz, primary) + Lenovo L24q-30 (1440p, rotated portrait)
- **Installer:** [JaKooLit/Debian-Hyprland](https://github.com/JaKooLit/Debian-Hyprland)

## Hyprland Config Structure

The JaKooLit installer separates configs into two layers — **default configs** (updated by the installer) and **user configs** (yours to edit, never touched by updates).

```
config/hypr/
├── hyprland.conf          # Main entry point — sources everything below
├── monitors.conf          # Monitor layout (generated by nwg-displays, do not hand-edit)
├── workspaces.conf        # Also managed by nwg-displays — use UserSettings.conf for rules
├── hypridle.conf          # Idle / screen timeout settings
├── hyprlock.conf          # Lock screen config (uses .wallpaper_current, matches active wallpaper)
├── hyprlock-1080p.conf    # Lock screen variant for 1080p
├── hyprlock-2k.conf       # Lock screen variant for 1440p
│
├── configs/               # DEFAULT configs — from the installer
│   ├── Keybinds.conf      # Built-in keybinds (don't edit; override in UserConfigs)
│   ├── SystemSettings.conf
│   ├── ENVariables.conf
│   ├── WindowRules.conf
│   ├── Startup_Apps.conf
│   └── Laptops.conf
│
├── UserConfigs/           # YOUR configs — safe to edit, not overwritten by updates
│   ├── 00-Readme          # Explanation from installer
│   ├── 01-UserDefaults.conf   # Default apps: $term, $files, $EDITOR, search engine
│   ├── UserSettings.conf      # Main Hyprland settings + workspace-to-monitor assignments
│   ├── UserKeybinds.conf      # Add/override keybinds here
│   ├── UserDecorations.conf   # Window decorations / blur / shadows
│   ├── UserAnimations.conf    # Animation config
│   ├── ENVariables.conf       # Your env variable overrides
│   ├── WindowRules.conf       # Your window rules
│   ├── Startup_Apps.conf      # Apps to autostart (also has cursor warp on startup)
│   ├── Laptops.conf           # Laptop-specific settings
│   └── LaptopDisplay.conf     # Laptop display config
│
├── UserScripts/           # YOUR scripts — safe to edit
│   ├── WallpaperSelect.sh     # Wallpaper picker — applies to ALL monitors
│   ├── WallpaperRandom.sh     # Random wallpaper — applies to ALL monitors
│   ├── WallpaperAutoChange.sh # Auto wallpaper rotation
│   ├── WallpaperEffects.sh    # Wallpaper shader effects
│   ├── RofiBeats.sh           # Music via Rofi
│   ├── Weather.sh / Weather.py # Weather widget
│   └── ...
│
├── scripts/               # INSTALLER scripts — from JaKooLit (don't edit)
│   ├── *.sh               # Hyprland utility scripts (brightness, volume, etc.)
│   └── keybinds_parser.py
│       # NOTE: dots-tui binary excluded from this repo (17MB compiled binary)
│       # Get it from the Debian-Hyprland installer
│
├── animations/            # Animation presets (selectable via Rofi)
├── wallust/               # Wallust color scheme templates for Hyprland
└── Monitor_Profiles/      # Saved monitor layout profiles
```

### Key Files to Customize

| File | What to change |
|---|---|
| `UserConfigs/01-UserDefaults.conf` | Default terminal, file manager, editor, search engine |
| `UserConfigs/UserKeybinds.conf` | Add your own keybinds (unbind first if replacing) |
| `UserConfigs/UserSettings.conf` | Hyprland settings + workspace-to-monitor assignments |
| `UserConfigs/Startup_Apps.conf` | Apps to launch on login |
| `UserConfigs/ENVariables.conf` | Nvidia, scaling, and other env vars |
| `monitors.conf` | Monitor layout — managed via `nwg-displays`, do not hand-edit |

### Monitor & Workspace Setup

| Monitor | Workspaces | Notes |
|---|---|---|
| Acer XV275K P3 (4K 160Hz) | 1 – 5 | Primary, cursor starts here on login |
| Lenovo L24q-30 (1440p portrait) | 6 – 10 | Secondary, rotated 90° |

Workspace rules live in `UserConfigs/UserSettings.conf` (safe from nwg-displays overwrites).

## Waybar

```
config/waybar/
├── config → configs/TOP-Default   # Active layout (symlink)
├── style.css → style/[Extra] Prismatic Glow.css  # Active style (symlink)
├── configs/        # Layout presets — controls module placement and bar position
├── style/          # CSS theme files — controls colors, shapes, transparency
├── Modules/        # Module definitions used by layouts
├── ModulesCustom/  # Custom modules
├── ModulesGroups/  # Grouped module definitions
├── ModulesVertical/ # Modules for vertical bars
├── ModulesWorkspaces/ # Workspace-specific modules
├── UserModules/    # User-added modules
└── wallust/        # Wallust color templates for Waybar (auto-generated on wallpaper change)
```

Switch layouts and styles interactively: `SUPER+ALT+B` (layout picker) / `SUPER+CTRL+B` (style picker).

### Layout presets (configs/)

Layouts control bar **position**, **modules shown**, and **arrangement**. Style is independent.

| Layout | Description |
|---|---|
| `TOP-Default` | Full-width top bar. Dense: cava visualizer, media player, window title left; workspaces center; clock, weather, tray, notifications right. |
| `TOP-Default-Laptop-glass` | Like Default but floating (margins on sides), designed to pair with glass styles. |
| `TOP-ddubs-simple-bar` | Floating pill bar with wide side margins. Media/player left, clock center, system right. macOS simple-bar inspired. |
| `TOP-Minimal-Short` | Narrow fixed-width bar (~1050px). Clock + weather left, roman numeral workspaces center, minimal right. |
| `TOP-Minimal-Long` | Full-width but very few modules. Clock left, workspaces center, tray right. |
| `TOP-Simple` | Balanced. Workspaces + tray + media left, clock center, system stats right. |
| `TOP-Simpliest` | Fewest modules of all top layouts. Just the essentials. |
| `TOP-Sleek` | Very thin (14px height), tight margins. Menu + workspaces + media left, clock center, tray right. |
| `TOP-Arrow` | Uses arrow/chevron separators between module groups. |
| `[TOP & BOT] SummitSplit` | Two-bar setup: top bar for system info, bottom bar for workspaces. |
| `[TOP & BOT] SummitSplit-glass` | Same as above but designed for glass/transparent styles. |
| `[BOT] Default` | Bottom-only bar, similar modules to TOP-Default. |
| `[BOT] Simple` / `[BOT] Sleek` | Bottom bar, minimal. |
| `[LEFT] WestWing` / `[RIGHT] EastWing` | Vertical side bars. Pair with VERTICAL style variants. |

### Style presets (style/)

Styles control **colors**, **transparency**, **border-radius**, and **visual weight**. Mix and match with any layout.

| Style | Description |
|---|---|
| `[Extra] Prismatic Glow` | *(current)* Colorful, glowing borders. Vibrant with wallust colors. |
| `[Transparent] Crystal Clear` | Fully transparent bar background. Floating pill-shaped modules with rounded corners. Very clean, macOS-like. |
| `[Wallust Transparent] Crystal Clear` | Same as above but tinted with wallust wallpaper colors. |
| `[Extra] Modern-Combined` | Opaque combined-pill style. Modules grouped into unified blocks. |
| `[Extra] Modern-Combined - Transparent` | Same shape as above, semi-transparent background. |
| `[WALLUST] ML4W-modern` | Clean and modern, transparent bar, rounded wallust-colored workspace pills. |
| `[Wallust] Simple` | Minimal pill modules, transparent bar background, thin border. |
| `[Wallust] Chroma Fusion` | Wallust colors with gradient module borders. |
| `[Dark] Golden Noir` | Dark bar with gold accent colors. |
| `[Dark] Half-Moon` | Dark style with half-moon shaped module pills. |
| `[Catppuccin] Mocha` | Fixed Catppuccin Mocha palette, ignores wallust colors. |
| `[Catppuccin] Latte` | Fixed Catppuccin Latte (light) palette. |
| `[Black & White] Monochrome` | Stark black and white, no color. |
| `[Rainbow] RGB Bordered` | Animated rainbow borders. |
| `[Retro] Simple Style` | Retro/flat look, no rounded corners. |

**macOS-like combinations to try:**
- Layout `TOP-ddubs-simple-bar` + Style `[Transparent] Crystal Clear` — closest to macOS menu bar feel
- Layout `TOP-Minimal-Short` + Style `[Wallust Transparent] Crystal Clear` — minimal floating bar
- Layout `[TOP & BOT] SummitSplit-glass` + Style `[Extra] Modern-Combined - Transparent` — menu bar + dock-like split

## Color Theming (Wallust)

[Wallust](https://codeberg.org/explosion-mental/wallust) generates color schemes from wallpapers and applies them across the desktop.

```
Pick wallpaper (SUPER+CTRL+ALT+W or CTRL+ALT+W for random)
        ↓
WallpaperSelect.sh applies it to ALL monitors via swww
        ↓
WallustSwww.sh runs wallust against the image
        ↓
Regenerates color variables for: Waybar, Rofi, Hyprland, Kitty, Ghostty
        ↓
Copies wallpaper to .wallpaper_current → lock screen updates automatically
        ↓
Waybar and terminals reload colors live
```

Templates live in each app's `wallust/` subdirectory:
- `config/hypr/wallust/` — Hyprland border/shadow colors
- `config/waybar/wallust/` — Waybar CSS color variables
- `config/rofi/wallust/` — Rofi color variables
- `config/wallust/` — Wallust config and template definitions

## Installing on a Fresh System

> **Prerequisites:** Run [JaKooLit's Debian-Hyprland installer](https://github.com/JaKooLit/Debian-Hyprland) first to get all dependencies, then overlay these dotfiles.

```bash
git clone https://github.com/najchris11/dotfiles.git ~/dotfiles
cd ~/dotfiles
bash install.sh
```

The install script backs up any existing configs and symlinks everything into place.
